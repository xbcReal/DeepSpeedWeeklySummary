🚀 DeepSpeed

v0.19.0
发布时间：2026年5月6日

本次发布的主要亮点：

  • [AutoSP] 多模态模型 (ViT + LLM) 序列并行支持
  • [Muon Optimizer] Gram Newton-Schulz 正交化支持
  • [NVTX] DeepSpeed NVTX domain 性能分析支持
  • [DeepCompile+Z3] 修复 PyTorch v2.9/2.10 兼容性
  • [Dynamic Offload] 与静态优化器 offload 兼容
  • [ZeRO-1/2] 修复 CPU-offloaded 梯度丢失问题
  • [BF16_Optimizer] 修复 ZeRO-1 下梯度泄漏
  • [MoE] 修复 topkgating 重大 bug
  • [FPQuantizer] 修复构建错误和 UB 警告
  • [WarmupCosineLR] 修复多参数组初始化

新贡献者：@alek6kun, @Cursx, @roycho96, @st-bang97, @maxyu1115, @excepshenal, @heurry

社区 Issue
#8001 - EvoformerAttention should auto-detect CUTLASS instead of requiring CUTLASS_PATH
提交者：@MaxTretikov | 2026-05-08 | 状态：Open
建议 EvoformerAttention 的 op builder 自动发现 CUTLASS 路径（类似 PyTorch 发现 CUDA 的方式），而不是强制用户手动设置 CUTLASS_PATH 环境变量。提交者同步提交了对应的 PR #8000。
#7998 - [REQUEST] Will ZeRO-3 support different module usage? enhancement
提交者：@Creator404 | 2026-05-07 | 状态：Open
功能请求：希望 ZeRO-3 能支持不同 rank 对同一模块的不同调用次数（动态控制流）。目前 ZeRO-3 要求每个模块在所有 rank 上被调用相同次数，否则会导致挂起。例如一个 LLM 模块 A 和任务头 B，B 在不同 rank 可能被调用不同次数。
#7991 - [BUG] Loss connection to ranks bug training
提交者：@Creator404 | 2026-05-04 | 状态：Open
在 2 节点 16 GPU (A100) 的训练中，backward 在第 9 步卡住。具体问题出在 scaled_global_norm() 计算中，pdb 可以获取所有 16 个 rank 的编号，但 self.bit16_groups 只返回 14 个结果后就断开连接。涉及 SAM3 与多模态模型的训练，只在 2 节点时复现。
Roadmap & 社区动态
🗺️ 版本路线与发展方向

v0.19.0 发布标志着 DeepSpeed 的重要里程碑：

  • AutoSP (自动序列并行) 正式落地，支持多模态模型 (ViT + LLM)，此前已合入 AutoSP 框架（PR #7860）
  • Muon 优化器生态持续完善 — Gram Newton-Schulz 正交化已合入，CPU offload (#7939) 和 ZeRO-3 兼容 (#7919) 已发布，博客文章 (#7962) 正在 review
  • AutoEP (自动 Expert 并行) 正在积极推进 — 核心 PR #7938 和 ZeRO-3 兼容 PR #7928 均在 review 中
  • SDMA 通信后端 — 来自 AMD 团队 @inkcherry 的 RFC (#7884) 和实现 (#7999) 都在推进中，旨在为 AMD GPU 提供基于 SDMA 的集体通信
  • TPU 支持 — torch_xla TPU 后端 (PR #7917) 持续 review 中，显示 DeepSpeed 正朝着更广泛的硬件支持方向发展
  • DeepCompile — 持续优化中，本次发布已修复与 PyTorch 2.9/2.10 的兼容性问题
📊 社区活跃度观察
  • v0.19.0 引入 7 位新贡献者，社区参与度持续增长
  • AMD GPU 相关的通信优化工作（SDMA）显示了 DeepSpeed 对多硬件平台的重视
  • Muon 优化器相关的 PR 数量较多，表明社区对该优化器有较高的关注和使用需求
  • MoE 相关优化（AutoEP、singleton MoE collectives）是当前的活跃开发方向之一
