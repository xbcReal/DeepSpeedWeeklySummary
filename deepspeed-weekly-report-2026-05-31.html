作者: @roycho96 | 合入: 2026-05-28

Feature #7992 Add engine.coalesce_grad_reduction() for ZeRO 1/2/3 multi-backward
作者: @roycho96 | 合入: 2026-05-28
新增 coalesce_grad_reduction() context manager，将 ZeRO 1/2/3 多次 backward 的 N 次 reduce-scatter 合并为 1 次。支持 GradCache 等 cached contrastive learning 场景（如 sentence-transformers CachedMultipleNegativesRankingLoss），显著降低通信开销。

👀 六、Review 中的 PR（重点关注）
本周新提交的 PR
Bugfix Review #8038 Normalize ZeRO-3 DeepCompile grad dtype before reduction
作者: @tohtana | 创建: 2026-05-30 | Reviewer: @tjruwase @loadams
修复 DeepCompile 在 ZeRO-3 中的梯度 dtype 不匹配问题。某些 backward kernel 产生 fp32 梯度，但 ZeRO-3 grad partition 期望 bf16，导致错误的通信 bucket 选择。此 PR 在 reduce 前对原始梯度做 dtype 归一化。
Bugfix Draft #8036 Fix DeepCompile ZeRO-1 grad target lifetime
作者: @tohtana | 创建: 2026-05-29 | Reviewer: @tjruwase @loadams @eternalNight
修复 DeepCompile ZeRO-1 中 reduce target buffer 生命周期过长的问题（compile-time buffer 存活到 optimizer step），导致梯度存储与 optimizer 临时存储重叠。在 8×H100 + Qwen3-8B 测试中峰值显存从 43.6 GiB 降至 39.4 GiB。
Feature Review #8035 Enable bf16 check_grad_overflow by default (matching fp16)
作者: @yongzhe-wang | 创建: 2026-05-29 | Reviewer: @tjruwase @tohtana
将 bf16 模式的 check_grad_overflow 默认值从 False 改为 True，与 fp16 行为对齐。解决了 ZeRO-2 + bf16 在特定条件下梯度溢出导致训练无声崩溃（NaN）的问题，影响 MoT 模型和异构 loss mask 场景。
CI Review #8034 Stop obsolete CI jobs on workflow cancellation
作者: @tohtana | 创建: 2026-05-28 | Reviewer: @loadams
在 workflow 被取消时自动停止过时的 CI job，节省计算资源。
Bugfix Review #8032 Fix DeepCompile ZeRO-3 release parameter lifetime
作者: @tohtana | 创建: 2026-05-28 | Reviewer: @tjruwase @loadams @eternalNight
修复 ZeRO-3 中 DeepCompile 参数释放后 padded buffer 仍然存活的内存泄漏问题。PR #7489 引入的 base/view 关系导致 release 只清理了 registry tensor 而未释放底层 StorageImpl。
Feature Draft #8031 [Draft] Add ZeRO-3 elastic checkpoint save/load support
作者: @nathon-lee | 创建: 2026-05-28
为 ZeRO-3 添加弹性 checkpoint 支持，允许以不同 DP world size 保存和恢复 checkpoint。解决抢占后在不同 GPU 数量恢复训练的需求，以及缩小硬件调试场景。ZeRO 1/2 已有此功能，此 PR 将其扩展到 Stage 3。
Feature Review #8027 Add On-Policy Distillation (OPSD) Trainer in DeepSpeed
作者: @PKUWZP | 创建: 2026-05-26 | Reviewer: @tohtana
新增 DeepSpeed 原生 on-policy 蒸馏训练器（~3.2k LOC，32 文件）。支持 forward-KL / reverse-KL / JSD 散度计算，student rollout → teacher scoring → student update 三阶段流水线。已在 2×H200 + Qwen2.5-0.5B/1.5B 上验证。配套 87 个 CPU 单元测试。

Q2 2026 Roadmap (#7861)

Roadmap issue 本周无新评论（最后更新: 2026-03-24 @tohtana 回复了关于 MoE 支持计划的问题），但从本周合入和 review 中的 PR 可以看到以下方向的显著推进：
